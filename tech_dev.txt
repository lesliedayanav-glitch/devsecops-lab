1. Instalar dependencias y probar local
En la carpeta del proyecto (SAMPLE_DEV_SEC_OPS):
npm install        # instala lodash, jest, etc.
npm test           # verifica que las pruebas pasen
npm run build      # crea la carpeta dist con el index.html

2. Inicializar Git y hacer el primer commit
git init
git add .
git commit -m "Initial DevSecOps demo"
git branch -M main

3. Crear el repo en GitHub y subirlo
Ve a GitHub → New repository → ponle el nombre (por ejemplo sample-dev-sec-ops).
NO inicialices con README (ya lo tienes local).
Luego, en tu terminal:
git remote add origin https://github.com/galarza-dev/sample_dev_sec_ops.git   # o por SSH
git push -u origin main


4. Verificar que el workflow se ejecute
En GitHub, entra a tu repo:
Pestaña Actions → debe aparecer el workflow DevSecOps Demo Pipeline.
Debería haberse disparado un run por el push a main.
Si todo está bien, verás los 3 jobs:
build_and_test
security_scan
deploy
Es probable que falle en security_scan por la versión de lodash → eso es perfecto para tu taller.

5. Activar GitHub Pages (para el CD)
En el repo en GitHub:
Settings → Pages
En “Source”, selecciona GitHub Actions (no branch).
Guarda.
La próxima vez que el pipeline pase completo, el job deploy publicará tu dist/ y te dará la URL.

6. (Opcional) Simular el “arreglo” de seguridad
Cuando quieras mostrar cómo “arreglas” el problema:
Cambia en package.json:
"lodash": "4.17.21" //4.17.21 , 4.17.19
Luego:
npm install
git add package.json package-lock.json
git commit -m "Fix lodash vulnerability"
git push

El pipeline se ejecuta de nuevo:
Pasa build_and_test
Pasa security_scan
Corre deploy y actualiza GitHub Pages.
